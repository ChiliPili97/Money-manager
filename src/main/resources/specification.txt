 Додаток для поанування бюджету.

Технологіїї:
Java 17 (or higher, but LTS version is preferable)
Spring 3 (Boot, JPA, Security)
PostgreSQL
Lombok
jjwt-api (io.jsonwebtoken)
Junit 5
Spring Boot Test

Вся комунікація повинна бути організована по REST API.

1. Всі ендпоінти мають бути доступні для користувача тільки після реєстрації, публічними є лише реєстрація та аутентифікація (JWT токен).

2. Категорії витрат, наприклад – Покупки, Машина, Послуги, Медицина, Сім’я – будь-які категорії на ваш вибір.
Організувати управління категоріями – можливість створити, подивитися, видалити категорію. Кожного разу, коли ви витратили якісь кошти,
їх потрібно записати до певної категорії витрат – реалізувати цю можливість. Якщо помилково додали кошти не в ту категорію,
додати можливість видалити ту саму суму. Якщо відчуваєте, що не вистачає якихось операцій в описі – додайте їх.

3. Вивести інформацію про користувача із використанням пагінації (тільки для аутентифікованих користувачів).
Відповідь від сервера має включати в себе дані про користувача та про категорії, до яких він додавав кошти.

4. Функціонал, який показує, скільки витрат було по заданим категоріям для певного користувача за певний період (наприклад тиждень, місяць і т.д.).
Спробуйте подумати, яка  інформація про витрати є необхідною і додати цей функціонал.

5. Весь написаний код повинен бути покритий тестами – окремі класи юніт-тестами, а API інтеграційними тестами.

6. Весь код повинен бути задокументований – javadoc.
